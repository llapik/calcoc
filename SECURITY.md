## 1. Модель угроз

Инструмент разработан как **локальный** — он запускается с загрузочной USB-флешки или
непосредственно на компьютере пользователя и не является публичным сервисом.

### Область применения

| Параметр | Значение |
|----------|---------|
| Аудитория | Владелец ПК / IT-специалист за физическим устройством |
| Сетевая доступность | Только `127.0.0.1` по умолчанию (не публичный порт) |
| Пользователи одновременно | Один |
| Хранение данных | Только на USB-накопителе или локальном диске |
| Передача данных наружу | Только по явному запросу (OpenRouter, см. раздел 5) |

### Угрозы, которые мы учитываем

| Угроза | Описание | Митигация |
|--------|----------|-----------|
| **Инъекция команд** | Вредоносный путь к устройству в параметрах repair-функций | Regex-валидация `/dev/…` перед передачей в subprocess |
| **Shell-инъекция через журнал** | Подмена `rollback_cmd` в SQLite-журнале | Команды хранятся как JSON-массивы; `shell=False` при выполнении |
| **Утечка API-ключа** | Ключ OpenRouter попадает в логи при ошибке | Логируется только последние 4 символа (`key_hint`) |
| **Фиксированный SESSION-ключ** | Статичный `SECRET_KEY` Flask позволяет подделку cookie | Ключ генерируется через `secrets.token_hex(32)` при каждом запуске |
| **Хранение PII в телеметрии** | Серийный номер материнской платы в открытом виде | Хранится усечённый SHA-256 (16 hex-символов) без возможности реверса |
| **Path traversal** | Относительные пути (`../../etc/passwd`) в backup/restore | Проверка `_validate_path()` в bootloader.py |

### Угрозы вне области применения

- Атаки на сетевой уровень (DDoS, MITM) — сервис не публичный.
- Атаки на браузер пользователя — интерфейс доступен только с localhost.
- Компрометация OpenRouter API — это третья сторона, ключ остаётся на вашем устройстве.

## 2. Архитектура безопасности

### Subprocess без оболочки

Все вызовы внешних утилит (`e2fsck`, `smartctl`, `dd`, `sfdisk`, `grub-install` и др.)
передаются как **список аргументов** — без `shell=True`. Это исключает интерпретацию
специальных символов оболочки (`; | && $(...)`).

Единственное исключение — команды-откаты из старых записей журнала (созданных до данной
версии), где выводится предупреждение и используется `shell=True` для обратной совместимости.

### Уровни риска (`safety_rules.yaml`)

Каждое деструктивное действие имеет уровень риска:

| Уровень | Цвет | Требует подтверждения | Требует бекапа | Примеры |
|---------|------|-----------------------|----------------|---------|
| `green`  | Зелёный | Нет | Нет | Чтение данных, диагностика |
| `yellow` | Жёлтый | Да | Да | Очистка временных файлов, исправление FS |
| `red`    | Красный | Да + повышенный допуск | Да | Запись MBR, переустановка GRUB |
| `black`  | Чёрный | Только экспертный режим | Обязательно | Прошивка BIOS (заблокировано по умолчанию) |

Код:

```python
from src.safety.classifier import SafetyClassifier
verdict = SafetyClassifier(config).check("fix_grub")
if not verdict.allowed:
    raise PermissionError(verdict.reason)
```

### Журнал и откат

Каждая операция, изменяющая данные, записывается в SQLite-журнал **до** выполнения.
Команда отката хранится как JSON-массив, что исключает инъекцию при восстановлении:

```json
["tar", "xzf", "/usb/backups/1234_etc_hosts", "-C", "/etc"]
```

## 3. Политика конфиденциальности

### Принципы

1. **Данные не покидают устройство** без явного действия пользователя.
2. **Минимальный сбор** — собирается только то, что нужно для диагностики.
3. **Анонимизация** — идентификаторы устройств хешируются перед сохранением.
4. **Прозрачность** — все данные доступны пользователю через веб-интерфейс.

### Типы хранимых данных

| Что | Где хранится | Форма хранения | Зачем |
|-----|-------------|----------------|-------|
| Аппаратные параметры (CPU, RAM, диски) | RAM (только в памяти сессии) | Объекты Python | Диагностика |
| Температуры, SMART-статус, загрузка | `telemetry.db` на USB | SQLite, анонимно | Предсказание отказов |
| Идентификатор машины | `telemetry.db` | SHA-256[16 hex] без реверса | Группировка истории |
| Журнал операций | `journal.db` на USB | SQLite | Откат действий |
| Резервные копии файлов/MBR | `backups/` на USB | Файлы | Восстановление |
| Логи приложения | `logs/calcoc.log` | Текст | Отладка |
| API-ключ OpenRouter | Только ENV/settings.yaml | Plaintext | Доступ к облачному AI |

### Что НЕ собирается

- Содержимое пользовательских файлов.
- Пароли, ключи шифрования, куки браузера.
- Сетевые пакеты или история соединений.
- Геолокация.
- Серийные номера в открытом виде (хешируются).

## 4. Что собирается и хранится

### Телеметрия (`telemetry.db`)

База данных хранится **только на USB-накопителе** и никуда не отправляется.

```
Поля: timestamp, machine_id (SHA-256[:16]), cpu_temp, cpu_usage,
      ram_usage_pct, disk_temps (JSON), smart_status (JSON),
      power_on_hours (JSON), gpu_temp
```

Пример записи:

```json
{
  "machine_id": "a3f7b2c1d4e5f609",
  "cpu_temp": 52.0,
  "ram_usage_pct": 61.3,
  "smart_status": {"/dev/sda": true}
}
```

### Журнал операций (`journal.db`)

Содержит историю всех repair-действий с командами отката. Файл находится на USB-накопителе
и не содержит пользовательских данных (только пути и системные команды).

### Логи (`calcoc.log`)

Уровень `INFO` по умолчанию. API-ключи в логи не попадают — при отладке выводится только
маскированный суффикс (`...abcd`). Логи не ротируются автоматически — при необходимости
очистите вручную.

## 5. Что передаётся наружу

По умолчанию ничего. Данные покидают устройство только в одном случае:

### OpenRouter (опционально)

Если пользователь настроил OpenRouter-ключ, AI-чат будет отправлять запросы на
`https://openrouter.ai/api/v1/chat/completions`.

**Что отправляется:**
- Текст вашего сообщения в чате.
- Системный prompt (инструкции для AI).
- Краткая сводка аппаратуры и проблем (если был выполнен `/scan`).

**Что НЕ отправляется:**
- Содержимое файлов.
- Серийные номера устройств.
- Пароли или ключи.
- Полный вывод `smartctl` (только summary_text).

Вы можете проверить точный состав запроса в `src/ai/prompts.py` и `src/ai/openrouter.py`.

### Обновление баз ClamAV (опционально)

Команда `freshclam` обращается к серверам ClamAV для получения обновлений сигнатур.
Запускается только при явном вызове (`update_databases()`).

## 6. Управление секретами

### API-ключ OpenRouter

**Рекомендуемый способ** — переменная окружения (ключ не попадает в файлы конфигурации):

```bash
export OPENROUTER_API_KEY="sk-or-..."
python -m src.core.app
```

**Альтернатива** — `config/settings.yaml` (файл должен быть недоступен другим пользователям):

```yaml
openrouter:
  api_key: "sk-or-..."
```

```bash
chmod 600 config/settings.yaml
```

**Никогда не коммитьте ключ в git.** Файл `.gitignore` уже содержит паттерны для защиты:

```
config/settings.yaml   # если хотите хранить ключ в файле — добавьте в .gitignore
*.key
.env
```

### Flask SECRET_KEY

Генерируется через `secrets.token_hex(32)` при каждом запуске — не является постоянным.
Для production-окружений (если понадобится) задайте через `SECRET_KEY` в env.

## 7. Система отката и резервных копий

Перед любой деструктивной операцией создаётся резервная копия:

| Тип | Инструмент | Где хранится | Как восстановить |
|-----|-----------|-------------|-----------------|
| Файл | `shutil.copy2` | `backups/<timestamp>_<name>` | `cp -p` |
| Директория | `tar czf` | `backups/<timestamp>.tar.gz` | `tar xzf` |
| MBR (512 байт) | `dd` | `backups/<timestamp>.mbr` | `dd if=... bs=446` |
| Таблица разделов | `sfdisk --dump` | `backups/<timestamp>.sfdisk` | `sfdisk < file` |

Журнал операций позволяет откатить последнее действие:

```python
from src.rollback.backup import BackupManager
mgr = BackupManager("/mnt/usb_data/backups", journal)
mgr.rollback_last()
```

Или через веб-интерфейс: **Журнал → Откатить последнее действие**.

### Важно: MBR-бекап

Восстановление MBR перезаписывает только первые **446 байт** (bootstrap-код),
оставляя таблицу разделов (байты 446–512) нетронутой. Это сделано намеренно,
чтобы не повредить разметку диска при откате.

## 8. Известные ограничения

| # | Ограничение | Обходной путь |
|---|-------------|--------------|
| 1 | Flask работает без HTTPS | Используется только на localhost; добавьте reverse-proxy с TLS для удалённого доступа |
| 2 | Нет rate-limiting в API | Локальный инструмент — не предполагается публичный доступ |
| 3 | `rollback_cmd` для старых записей журнала выполняется с `shell=True` | Новые записи используют JSON-массивы; обновите устаревший журнал при необходимости |
| 4 | Логи не ротируются автоматически | Очищайте `logs/calcoc.log` вручную или настройте logrotate |
| 5 | `settings.yaml` хранит ключ plaintext | Используйте переменную окружения `OPENROUTER_API_KEY` |

## 9. Сообщить об уязвимости

Если вы обнаружили уязвимость:

1. **Не** создавайте публичный issue.
2. Откройте [приватный security advisory](https://github.com/llapik/calcoc/security/advisories/new)
   в репозитории GitHub.
3. Укажите:
   - Тип уязвимости (инъекция, утечка данных и т.д.)
   - Шаги воспроизведения
   - Возможный impact
   - Предлагаемое исправление (опционально)

Ответ в течение 72 часов. Публичное раскрытие — после выпуска исправления.
