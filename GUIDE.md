## 1. Что это такое и зачем?

**AI PC Repair & Optimizer** — это инструмент на флешке, который:

| Умеет | Пример |
|-------|--------|
| Проверить «здоровье» компьютера | «Жёсткий диск скоро сломается — сделайте копию данных» |
| Найти проблемы | «Температура процессора 92°C — нужна чистка» |
| Дать совет по улучшению | «Замените HDD на SSD — загрузка ускорится с 60 до 12 секунд» |
| Поговорить с AI на русском | «Почему мой ПК стал медленным?» |
| Исправить ошибки | Ошибки файловой системы, вирусы, засорение |

Всё это работает **без интернета**, **прямо с флешки**, на любом ПК от старого Pentium 4 до
современного Core i9.

## 2. Что вам понадобится

### Для флешки (Способ А)
- Флешка USB 3.0 объёмом **256 ГБ** или больше
- Компьютер с Linux (или Linux VM) для записи флешки

### Для запуска на своём ПК (Способ Б — только разработчикам)
- Компьютер с Linux/macOS/WSL
- Python **3.10** или новее

### Проверить версию Python

Откройте терминал (в Windows — WSL или Git Bash) и введите:

```
python3 --version
```

Вы должны увидеть что-то вроде `Python 3.11.2`. Если написано `3.9` или старше — обновите Python.

## 3. Способ А — Использовать как загрузочную флешку (рекомендуется)

### Шаг 1. Скачайте или соберите проект

```bash
# Откройте терминал и выполните:
git clone https://github.com/llapik/calcoc.git
cd calcoc
```

### Шаг 2. Подготовьте флешку

> ⚠️ **ОСТОРОЖНО!** Следующая команда **сотрёт все данные** на флешке.
> Убедитесь, что указали правильное устройство!

Узнайте имя вашей флешки:

```bash
lsblk
```

Вы увидите список дисков. Флешка обычно называется `/dev/sdb` или `/dev/sdc`.
Ищите устройство с нужным размером (256 ГБ):

```
NAME   SIZE TYPE MOUNTPOINT
sda    500G disk             ← это ваш основной диск (НЕ трогайте!)
sdb    256G disk             ← это флешка (её и используем)
```

### Шаг 3. Запишите программу на флешку

```bash
# Замените /dev/sdb на имя вашей флешки!
sudo bash scripts/install_to_usb.sh /dev/sdb
```

Программа спросит подтверждение — введите `YES` (заглавными буквами).

### Шаг 4. Скопируйте AI-модель (опционально)

Без AI-модели программа работает в режиме «только диагностика».
Для полноценного AI-чата скопируйте GGUF-файл модели в папку `models` на флешке:

```bash
# Флешка смонтируется автоматически. Найдите её:
ls /media/$USER/

# Скопируйте модель (замените путь на свой):
cp ~/Downloads/phi-3-mini-4k-instruct.Q4_K_M.gguf /media/$USER/AIREPAIR_DATA/models/
```

**Где взять модель?** — см. раздел [FAQ](#11-частые-вопросы-faq).

### Шаг 5. Загрузитесь с флешки

1. Вставьте флешку в **ремонтируемый** компьютер
2. Включите компьютер и сразу нажимайте `F12` (или `F2`, `Delete`, `Esc` — зависит от материнской
   платы), пока не появится меню загрузки
3. Выберите вашу флешку в списке
4. Появится меню GRUB — выберите **«Normal Mode»**
5. Дождитесь загрузки (1–2 минуты)
6. Откройте браузер и перейдите на `http://127.0.0.1:8080`

```
┌─────────────────────────────────────────────┐
│  AI PC Repair & Optimizer — Boot Menu       │
│                                             │
│  > Normal Mode           ← выберите это    │
│    Safe Mode (no AI)                        │
│    Expert Mode                              │
│    Rollback Last Session                    │
│    Boot from local disk                     │
└─────────────────────────────────────────────┘
```

## 4. Способ Б — Запустить прямо на своём компьютере

Этот способ подходит для **разработчиков** или если хотите просто попробовать программу.

### Шаг 1. Получите исходный код

```bash
git clone https://github.com/llapik/calcoc.git
cd calcoc
```

### Шаг 2. Настройте окружение (одна команда)

```bash
bash scripts/setup_env.sh
```

Эта команда автоматически:
- Создаст изолированное Python-окружение
- Установит все необходимые библиотеки
- Создаст тестовую базу знаний

**Ожидаемый вывод в конце:**
```
=== Setup complete! ===
Activate the environment:  source .venv/bin/activate
Run the application:       python -m src.core.app
```

### Шаг 3. Активируйте окружение

```bash
source .venv/bin/activate
```

После этого в начале строки появится `(.venv)` — это значит, окружение активно:
```
(.venv) user@computer:~/calcoc$
```

### Шаг 4. Запустите приложение

```bash
python -m src.core.app
```

Вы увидите:
```
2026-03-01 12:00:00 [INFO] calcoc.app: Starting web server on 127.0.0.1:8080
```

### Шаг 5. Откройте браузер

Перейдите по адресу: **http://127.0.0.1:8080**

Готово! Программа работает.

- Операционные системы на дисках

Диагностика займёт **30–90 секунд**.

## 8. OpenRouter — AI через интернет

Если у вашего ПК меньше 4 ГБ RAM или нет GGUF-модели, используйте **OpenRouter** —
бесплатный (с лимитами) AI через интернет.

### Получить API-ключ (бесплатно)

1. Откройте браузер и зайдите на **https://openrouter.ai**
2. Нажмите **Sign Up** (регистрация)
3. Создайте аккаунт (можно через Google)
4. Перейдите в раздел **Keys** → **Create Key**
5. Скопируйте ключ (начинается с `sk-or-...`)

### Добавить ключ в программу

**Способ 1 — через настройки интерфейса:**
1. Нажмите ⚙️ в правом верхнем углу
2. В поле «AI-бэкенд» выберите «OpenRouter (облако)»

**Способ 2 — через переменную окружения (надёжнее):**

```bash
# Добавьте строку в ~/.bashrc или ~/.zshrc:
export OPENROUTER_API_KEY="sk-or-ваш-ключ-здесь"

# Затем перезапустите программу:
source ~/.bashrc
python -m src.core.app
```

**Способ 3 — в файле config/settings.yaml:**

```yaml
openrouter:
  api_key: "sk-or-ваш-ключ-здесь"  # вставьте ваш ключ

ai:
  backend: "openrouter"             # переключить на OpenRouter
```

## 9. Уровни безопасности — когда что нажимать

Каждое действие в программе имеет цветовую метку риска:

```
🟢 ЗЕЛЁНЫЙ — безопасно
   Только чтение данных. Ничего не изменяется.
   Примеры: диагностика, просмотр S.M.A.R.T., анализ логов.
   → Можно делать смело, в любое время.

🟡 ЖЁЛТЫЙ — умеренный риск
   Изменения, которые можно откатить. Автоматически создаётся бэкап.
   Примеры: очистка временных файлов, исправление файловой системы.
   → Программа попросит подтверждение. Нажмите «Да» если уверены.

🔴 КРАСНЫЙ — высокий риск
   Серьёзные изменения. Могут повлиять на загрузку ОС.
   Примеры: восстановление загрузчика, изменение разделов.
   → Делайте ТОЛЬКО если понимаете что делаете. Обязателен бэкап.

⚫ ЧЁРНЫЙ — заблокировано
   Слишком опасно без специальной подготовки.
   Примеры: перепрошивка BIOS, полное стирание диска.
   → Заблокировано по умолчанию. Доступно только в режиме эксперта.
```

### Как включить режим эксперта

> ⚠️ Включайте только если точно знаете, что делаете!

В настройках ⚙️ поставьте галочку **«Режим эксперта»**.

## 10. Откат изменений — если что-то пошло не так

Перед каждым изменением программа **автоматически** создаёт резервную копию.
Если что-то пошло не так:

### Через веб-интерфейс
1. Перейдите на вкладку **📋 Журнал**
2. Найдите последнее действие
3. Нажмите **«Откатить»**

### Через меню загрузки флешки
1. Перезагрузите компьютер с флешки
2. В меню GRUB выберите **«Rollback Last Session»**
3. Программа автоматически отменит последнее изменение

### Через командную строку
```bash
python3 -c "
from src.rollback.backup import BackupManager
from src.rollback.journal import Journal
j = Journal('/mnt/usb_data/journal.db')
bm = BackupManager('/mnt/usb_data/backups', j)
result = bm.rollback_last()
print('Откат выполнен!' if result else 'Нечего откатывать')
"
```

## 11. Частые вопросы (FAQ)

### ❓ Где скачать AI-модели (GGUF-файлы)?

Бесплатно на HuggingFace. Вот рекомендуемые по объёму ОЗУ:

| Объём ОЗУ | Модель | Размер файла | Ссылка |
|-----------|--------|--------------|--------|
| 2–4 ГБ | SmolLM 135M | ~90 МБ | [HuggingFace](https://huggingface.co/HuggingFaceTB/SmolLM-135M-Instruct-GGUF) |
| 4–8 ГБ | Phi-3 Mini 4K | ~2.2 ГБ | [HuggingFace](https://huggingface.co/microsoft/Phi-3-mini-4k-instruct-gguf) |
| 8–16 ГБ | LLaMA 3 8B Q4 | ~4.7 ГБ | [HuggingFace](https://huggingface.co/bartowski/Meta-Llama-3-8B-Instruct-GGUF) |
| 16+ ГБ | LLaMA 3 8B Q6 | ~6.1 ГБ | [HuggingFace](https://huggingface.co/bartowski/Meta-Llama-3-8B-Instruct-GGUF) |

Скачайте файл с расширением `.gguf` и скопируйте в папку `models/` на флешке или в директорию
из `config/settings.yaml` → `paths.models_dir`.

### ❓ Диагностика ничего не находит

Программа использует стандартные Linux-утилиты. Убедитесь, что установлены:
```bash
# Проверить наличие
which smartctl   # диагностика дисков
which dmidecode  # информация о железе
which lspci      # информация о видеокарте

# Установить (Ubuntu/Debian):
sudo apt install smartmontools dmidecode pciutils
```

### ❓ AI не отвечает / пишет «модель не загружена»

Три варианта:
1. **Нет GGUF-файла** — скачайте модель (см. выше)
2. **Мало ОЗУ** — используйте OpenRouter (раздел 8)
3. **Неверный путь** — проверьте `config/settings.yaml`, строка `models_dir`

### ❓ Как узнать клавишу входа в BIOS для своей материнской платы?

При включении компьютера в правом нижнем углу обычно написана клавиша.
Также можно поискать в интернете модель своей материнки + «boot menu key».

Типичные клавиши:
- **F12** — большинство ноутбуков и современных материнок
- **F2** — ASUS, многие ноутбуки
- **Delete** — многие настольные материнки
- **F10** — HP
- **F1** — Lenovo ThinkPad
- **Esc** — некоторые ноутбуки

### ❓ Программа говорит «диск скоро сломается» — что делать?

**Немедленно** сделайте резервную копию важных данных!

```
1. Скопируйте фото, документы, музыку на другой диск или в облако
2. Только ПОСЛЕ этого — думайте о ремонте
3. Если диск ещё работает — купите новый и перенесите данные
```

### ❓ Как остановить программу?

В терминале нажмите `Ctrl + C`.

### ❓ Как обновить антивирусные базы ClamAV?

```bash
# При наличии интернета:
sudo freshclam
```

## 12. Если что-то сломалось

### Программа не запускается

```bash
# Проверьте Python:
python3 --version   # нужно 3.10+

# Пересоздайте окружение:
rm -rf .venv
bash scripts/setup_env.sh
source .venv/bin/activate
python -m src.core.app
```

### Ошибка «No module named 'flask'»

```bash
source .venv/bin/activate    # активируйте окружение!
pip install -r requirements.txt
```

### Страница не открывается в браузере

1. Убедитесь, что программа запущена (видите строку `Starting web server`)
2. Адрес должен быть **http** (не https): `http://127.0.0.1:8080`
3. Попробуйте другой браузер

### Все тесты разом (проверка установки)

```bash
source .venv/bin/activate
pytest tests/ -v
```

Если все тесты зелёные (`PASSED`) — установка прошла корректно.

### Посмотреть логи программы

```bash
# Логи выводятся в терминал и в файл:
cat /mnt/usb_data/logs/calcoc.log    # на флешке
# или при разработке:
tail -f /tmp/calcoc/logs/calcoc.log
```
